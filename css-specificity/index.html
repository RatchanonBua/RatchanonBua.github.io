CSS Specificity

    CSS Specificity คืออะไร?

  กฎในการเลือก Selector ที่ซึ่งจัดลำดับความสำคัญของ Selector  

  การกำหนด Selector แต่ละประเภท ให้มี Priority ที่แตกต่างกันออกไป  

  Selector ที่มีความสำคัญมาก จะถูกเลือกมา Apply กับ Element ก่อน  

  CSS Specificity จะจัด Priority ของ CSS Selector เพื่อการเรียกใช้งาน  

  CSS Specificity ยังรวมการจัด Priority ของวิธีการ Insert CSS อีกด้วย  

----

 ลำดับความสำคัญของ Selector

เรียงจากมากที่สุดไปยังน้อยที่สุด ตามลำดับ

+ Inline Style

+ Internal Style Sheet

- ID Selector

- Class Selector

- Element Selector

+ External Style Sheet

- ID Selector

- Class Selector

- Element Selector

----

  ประเภทของ CSS Selector

 1. Element Selector (element-name)

การเลือก Apply CSS เข้าไปที่ Element โดยตรง

 2. Class Selector (.class-name)

การเลือก Apply CSS เข้าไปที่ Element โดยการเลือก Class ของ Element และทำการ Apply Element ทั้งหมดใน Class

 3. ID Selector (#id-name)

การเลือก Apply CSS เข้าไปที่ Element โดยการเลือก ID ของ Element และทำการ Apply Element ทั้งหมดที่อยู่ใน ID

----

  วิธีการ Insert CSS ลงใน Element

 1. External Style Sheet

การ Import ไฟล์ Stylesheet นำมาใช้ Apply Element ของ HTML

 2. Internal Style Sheet

การเขียน Stylesheet ที่ไว้ใช้ Apply Element ใน Tag Head ของ HTML

 3. Inline Styles

การเขียน Style กำหนดรูปแบบใน Tag ของ HTML เลย

----

CSS Specificity: Rules

Specificity เท่ากัน (ลำดับชั้นของ Selector เท่ากัน): ถือเอา Style ล่าสุด

ถ้าหากมี Specific เท่ากัน Browser จะนับเอา Style ล่าสุดมา Apply ลงใน HTML

ตัวอย่าง"ul li.active" ในไฟล์ "style-1.css" และ "ul li.active" ในไฟล์ "style-2.css" ถูกไฟล์ "home.html" เรียกขึ้นมา

แต่ว่าไฟล์ "style-1.css" ถูกลำดับการเรียกหลังจากเรียกไฟล์ "style-2.css"

ดังนั้นไฟล์ "home.html" จะใช้ Style ของไฟล์ "style-1.css" เพราะ กฎการใช้ Style ล่าสุดเมื่อมี Specific เท่ากัน

ID Selector จะมี Specific สูงกว่า Attribute Selector

ID Selector จะถูกเรียกก่อน Selector ตัวอื่นๆ เพราะ มี Specific ที่สูงกว่า ยกเว้น !important และ Inner Style

ตัวอย่าง

#rules {background-color: lightgreen;}

.rules {background-color: yellow;}

ดังนั้นBackground จะเป็นสีเขียวอ่อน

Contextual Selector หรือ Selector ที่เจาะจงกว่า จะมี Specific มากกว่า Element Selector ตัวเดียว

Selector ที่มีรายละเอียดมากกว่า หรือ เจาะจงกว่า Selector ตัวอื่นๆ จะถูกเรียกใช้ก่อน

ตัวอย่าง

.intro h2 { color: white; }

h2 { color: blue; }

ดังนั้นคลาส "intro" Header ระดับ 2 จะมีสีขาว ส่วน Header ระดับ 2 ที่ไม่ได้อยู่ในคลาส "intro" จะมีสีน้ำเงิน

Class Selector จะมี Specific สูงกว่า Element Selector

Class Selector จะถูกเรียกก่อน Element Selector

ตัวอย่าง

.intro { background-color: yellow; }

h1 {background-color: red;}

.

.

.

<h1 class="intro">This is a heading</h1>

ดังนั้นพื้นหลังของ Header ขั้นที่ 1 จะมีสีเหลือง

Universal Selector (ได้แก่ *) จะมีค่า Specificity เป็น 0

CSS Specificity: Calculate

จะคำนวณ Specificity ได้อย่างไร?

Selector แต่ละตัวจะมี Specific อยู่ค่าหนึ่ง Selector ที่อยู่ลำดับชั้นเดียวกันจะมี Specific เท่ากัน โดยวิธีการคำนวณจะใช้กฎ ดังนี้

Inline Style จะมี Specific อยู่ที่1-0-0-0หรือจะใช้จำนวน1000เพื่อให้ง่ายต่อการคำนวณ

ID Selector จะมี Specific อยู่ที่0-1-0-0หรือจะใช้จำนวน100เพื่อให้ง่ายต่อการคำนวณ

Class Selector จะมี Specific อยู่ที่0-0-1-0หรือจะใช้จำนวน10เพื่อให้ง่ายต่อการคำนวณ

Element Selector จะมี Specific อยู่ที่0-0-0-1หรือจะใช้จำนวน1เพื่อให้ง่ายต่อการคำนวณ

ตัวอย่างการคำนวณ Specificity

"ul li.active a" -> มีค่า Specific อยู่ที่ 13 (ul = 1, li = 1, .active = 10 และ a = 1)

"ul li a" ---------> มีค่า Specific อยู่ที่ 3 (ul = 1, li = 1 และ a = 1)

ดังนั้น Browser จะเลือก "ul li.active a" มา Apply เพราะ มี Specific มากกว่า "ul li a"
